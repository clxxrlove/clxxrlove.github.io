Universal Link 기반 초대 플로우
  • 문제 초대 링크를 받았을 때 앱 설치 여부에 따라 진입 흐름이 달라지고, 설치 후 다음 단계로 이어지지 않을 수 있었습니다.
  • 해결 서버에 AASA를 적용해 Universal Link 기반 초대 플로우를 구현하고, 링크 탭 시 앱이 설치돼 있으면 실행하고 없으면 App Store로 이동하도록 구성했습니다.
  • 결과 초대 링크를 통해 앱으로 진입한 뒤 커플 연결과 이후 정보 입력 플로우를 이어서 진행할 수 있게 했습니다.
FCM 라우팅과 알림 진입 정리
  • 문제 찌르기, 목표 완료, 반응처럼 알림 유형이 늘어나면서 알림별 화면 진입 규칙을 따로 관리하기 어려웠습니다.
  • 해결 twix://notification/[type] 형태의 딥링크를 enum으로 매핑하고 Coordinator에서 화면 전환을 처리했습니다. 알림 센터에는 lastId 기반 커서 페이지네이션과 IdentifiedArray를 적용했습니다.
  • 결과 알림 타입이 늘어나도 진입 규칙과 목록 상태를 한 구조 안에서 관리할 수 있게 했습니다.
연결 대기와 온보딩 상태 설계
  • 문제 상대방 연결 완료를 기다리는 구간과 이미 입력한 정보를 다시 받는 구간이 불편 요소였습니다.
  • 해결 연결 대기 화면은 polling으로 상태를 확인해 즉시 다음 단계로 넘기고, 온보딩은 coupleConnection → profileSetup → anniversarySetup → completed 상태를 기준으로 완료된 입력을 건너뛰도록 구성했습니다.
  • 결과 상대방이 먼저 입력한 정보는 다시 받지 않도록 처리해 중복 입력 단계를 덜었습니다.
TCA + MFA 구조 정리
  • 문제 전역 상태 중심의 TCA와 모듈 분리를 추구하는 MFA를 함께 쓰면서 화면 전환과 모듈 의존성이 복잡해졌습니다.
  • 해결 Interface / Implementation 분리, ViewFactory 패턴, Coordinator Reducer를 두고 상위 모듈이 하위 뷰 구현을 직접 알지 않도록 구성했습니다.
  • 결과 화면 전환과 모듈 경계를 분리해 기능 추가 시 의존성을 줄였습니다.

Structured Concurrency 기반 병렬 처리
약 4배 개선
  • 문제 음성 데이터 가공 과정이 순차적으로 실행되면서 응답 시간이 길었습니다.
  • 해결 프롬프트 엔지니어링으로 생성형 AI 답변 길이와 형식을 제한하고, Structured Concurrency 기반 병렬 처리로 전환했습니다. 응답을 기다리는 동안에는 애니메이션 기반 로딩 뷰도 설계했습니다.
  • 결과 최악 기준 약 4배 수준으로 줄였고, 평균 응답 시간은 15.5초에서 3.9초로 낮췄습니다.
PKCE + Keychain 인증 처리
  • 문제 외부 서비스 연동을 위해 인증 구조와 토큰 저장 방식을 정리할 필요가 있었습니다.
  • 해결 OAuth 2.1 PKCE 흐름을 구현하고, Keychain 기반 Token Storage 로직으로 토큰을 관리했습니다.
  • 결과 외부 서비스 연동에 필요한 인증 절차와 토큰 저장 구조를 앱 안에서 분리했습니다.
프롬프트 인젝션 대응
  • 문제 악의적 음성 데이터 입력 시 프롬프트 명령 인젝션 가능성이 있었습니다.
  • 해결 입력값 길이 제한과 래핑을 적용하고, 시스템 프롬프트와 사용자 입력 값을 분리하는 정책을 적용했습니다.
  • 결과 악성 입력이 프롬프트 명령으로 해석되지 않도록 차단 조치를 반영했습니다.
외부 API 의존성 조정
  • 문제 감정에 따른 음악 추천 기능을 Spotify API 기준으로 설계했지만, 개발 중 추천 API 지원 종료가 발생했습니다.
  • 해결 관련 기능을 제거하고, 핵심 기능을 외부 서비스에 의존할 때 생기는 리스크를 다시 점검했습니다.
  • 결과 이후 API를 고를 때 대체 가능성과 백업 플랜을 함께 보게 됐습니다.

  • 결과 후처리를 통해 OCR 인식 텍스트 품질을 개선했습니다.

  • 해결 APNs와 FCM 환경 설정, 토큰 수명주기, 권한 상태 변화를 함께 고려해 서버와 동기화하는 구조를 구현했습니다.
  • 결과 권한 상태 변화와 토큰 갱신 이벤트를 서버와 동기화할 수 있게 됐습니다.
